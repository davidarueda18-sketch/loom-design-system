# Loom Design System — Agnostic Web Component Adapter Generator

Generates professional Custom Element adapters that bridge Vanilla Extract atomic classes to framework-agnostic Web Components consumable from Angular and Vue.

## When to Use

- User asks to create a Web Component adapter for a design system primitive
- User asks to refactor an existing adapter to comply with the five laws
- User asks to add reactivity (getters/setters) to a Custom Element

---

## Five Mandatory Laws

### Law 1 — Zero Inline Styles
`this.style.*` is **forbidden**. Attributes must map to atomic CSS classes generated by Vanilla Extract (exported from the component's `.css.ts` file). If the `.css.ts` file does not export a class map object (e.g. `export const padding = styleVariants(...)`) for the token being mapped, **stop and generate that mapping first** before writing the adapter.

```ts
// ✅ CORRECT — class-based
const nextClass = key in styles.padding ? styles.padding[key as SpacingTokenKey] : null;
if (nextClass) this.classList.add(nextClass);

// ❌ WRONG — inline style
this.style.padding = spacingVars[key];
```

### Law 2 — Dual Reactivity (Attributes + Properties)
Every observed attribute must have a corresponding getter/setter so the element is reactive both to:
- **HTML attributes / Vue bindings** → via `observedAttributes` + `attributeChangedCallback`
- **JS property assignment / Angular bindings** → via getters and setters

Both paths must converge on `_sync()`.

```ts
// Setter bridges Angular property → attribute → _sync()
set padding(val: SpacingTokenKey | null) {
  if (val == null) this.removeAttribute('padding');
  else this.setAttribute('padding', val);
}

// Getter reads from the DOM (single source of truth)
get padding(): SpacingTokenKey | null {
  return this.getAttribute('padding') as SpacingTokenKey | null;
}
```

### Law 3 — Token Validation
Before applying a class derived from an attribute value, validate that the value exists as a key in the class map. Never cast blindly. Use a fallback or a no-op if the token is unknown.

```ts
// ✅ Validated
const nextClass = key != null && key in styles.padding
  ? styles.padding[key as SpacingTokenKey]
  : null;

// ❌ Unvalidated — will throw or apply wrong class
const nextClass = styles.padding[key as SpacingTokenKey];
```

### Law 4 — Idempotent _sync()
`_sync()` must track the previously applied class per property group and use `classList.replace` (or remove/add) to avoid accumulating stale classes on repeated calls. Never call `classList.add` without removing the previous class first.

```ts
// Track per-property previous class
private _prevClasses: Record<string, string | null> = {
  padding: null,
  paddingX: null,
  paddingY: null,
};

private _apply(
  prop: string,
  key: string | null,
  classMap: Record<string, string>,
): void {
  const next = key != null && key in classMap ? classMap[key] : null;
  const prev = this._prevClasses[prop] ?? null;
  if (next === prev) return;
  if (prev) this.classList.remove(prev);
  if (next) this.classList.add(next);
  this._prevClasses[prop] = next;
}
```

### Law 5 — TypeScript Integration
Always emit a `declare global` block with `HTMLElementTagNameMap` so consumers get full type checking and autocomplete when using `document.createElement('loom-*')` or JSX.

```ts
declare global {
  interface HTMLElementTagNameMap {
    'loom-box': LoomBox;
  }
}
```

## Required CSS Mapping Pattern

Before writing the adapter, the `.css.ts` file **must** export atomic class maps using `styleVariants`. This avoids any if/else logic inside the Web Component.

```ts
// Box.css.ts
import { style, styleVariants } from '@vanilla-extract/css';
import { spacingVars } from '../../../tokens/spacing/spacing.tokens.css.ts';

export const root = style({ boxSizing: 'border-box' });

// One atomic class per token per property — no if/else needed in the element
export const padding  = styleVariants(spacingVars, (val) => ({ padding:       val }));
export const paddingX = styleVariants(spacingVars, (val) => ({ paddingInline: val }));
export const paddingY = styleVariants(spacingVars, (val) => ({ paddingBlock:  val }));
```

Then the adapter only imports and indexes:

```ts
import * as styles from '../Box.css.ts';
// styles.padding['md'] → the atomic class string for padding: var(--spacing-md)
```

## Full Adapter Template

```ts
import * as styles from '../[Name].css.ts';
import type { [TokenKey] } from '../../../../tokens/index.ts';

class Loom[Name] extends HTMLElement {
  // ─── Observed attributes (drives HTML / Vue reactivity) ────────────────────
  static observedAttributes = ['[attr-1]', '[attr-2]'] as const;

  // ─── Getters / Setters (drives Angular / JS property reactivity) ───────────
  get [prop1](): [TokenKey] | null {
    return this.getAttribute('[attr-1]') as [TokenKey] | null;
  }
  set [prop1](val: [TokenKey] | null) {
    if (val == null) this.removeAttribute('[attr-1]');
    else this.setAttribute('[attr-1]', val);
  }

  // ─── Lifecycle ─────────────────────────────────────────────────────────────
  connectedCallback() {
    this.classList.add(styles.root);
    this._sync();
  }

  attributeChangedCallback() {
    this._sync();
  }

  // ─── State tracking (idempotent _sync) ─────────────────────────────────────
  private _prev: Record<string, string | null> = {
    [prop1]: null,
    [prop2]: null,
  };

  // ─── Sync logic ────────────────────────────────────────────────────────────
  private _sync(): void {
    this._apply('[prop1]', this.getAttribute('[attr-1]'), styles.[classMap1]);
    this._apply('[prop2]', this.getAttribute('[attr-2]'), styles.[classMap2]);
  }

  private _apply(
    prop: string,
    key: string | null,
    classMap: Record<string, string>,
  ): void {
    const next = key != null && key in classMap ? classMap[key] : null;
    const prev = this._prev[prop] ?? null;
    if (next === prev) return;
    if (prev) this.classList.remove(prev);
    if (next) this.classList.add(next);
    this._prev[prop] = next;
  }
}

customElements.define('loom-[name]', Loom[Name]);

declare global {
  interface HTMLElementTagNameMap {
    'loom-[name]': Loom[Name];
  }
}

export { Loom[Name] };
```

## Workflow

1. **Read `.css.ts`** — Identify which class map objects are exported. If a required class map is missing, generate it using `styleVariants` before writing the adapter.
2. **Read `.types.ts`** — Identify the token key types to use in getters/setters.
3. **Write the adapter** following the template above.
4. **Verify exports** — Ensure the adapter is exported from the component's `index.ts` and from the package's `index.elements.ts` (if it exists).
5. **Invoke `story` skill** if the user wants Storybook documentation for the new Custom Element.

---

## Checklist Before Finalizing

- [ ] No `this.style.*` calls anywhere in the adapter
- [ ] `observedAttributes` declares every reactive attribute
- [ ] Every observed attribute has a matching getter and setter
- [ ] `_apply()` compares `next === prev` before touching the DOM
- [ ] Token keys are validated with `key in classMap` before indexing
- [ ] `declare global` + `HTMLElementTagNameMap` is present
- [ ] The `.css.ts` file exports `styleVariants` class maps for every token used
- [ ] Adapter is exported from `index.ts` / `index.elements.ts`
